Thank you for installing {{ .Chart.Name }}!

Your cert-manager DNS01 webhook solver for Bunny DNS (bunny.net) has been deployed.

=== Next Steps ===

1. Create a Secret containing your Bunny.net API access key in the namespace where
   your Issuer or ClusterIssuer will live:

   kubectl create secret generic bunny-credentials \
     --from-literal=accessKey=<YOUR_BUNNY_ACCESS_KEY> \
     -n cert-manager

2. Create a ClusterIssuer (or Issuer) referencing this webhook:

   apiVersion: cert-manager.io/v1
   kind: ClusterIssuer
   metadata:
     name: letsencrypt-prod
   spec:
     acme:
       server: https://acme-v02.api.letsencrypt.org/directory
       email: your-email@example.com
       privateKeySecretRef:
         name: letsencrypt-prod
       solvers:
         - dns01:
             webhook:
               solverName: bunny
               groupName: {{ .Values.groupName }}
               config:
                 apiSecretRef:
                   name: bunny-credentials
                   key: accessKey

3. Request a certificate:

   apiVersion: cert-manager.io/v1
   kind: Certificate
   metadata:
     name: my-cert
   spec:
     secretName: my-cert-tls
     dnsNames:
       - "example.com"
       - "*.example.com"
     issuerRef:
       name: letsencrypt-prod
       kind: ClusterIssuer

For more information, see:
https://github.com/cvandesande/cert-manager-webhook-bunny
